# Proactive Refactoring

- If you identify code smells, architectural coupling, or opportunities for design improvement, you must propose the refactoring to the Developer. Do not execute refactoring without explicit approval.
- Once a refactoring or improvement proposal is approved by the Developer, you must update and synchronize both the technical documentation (files in `/docs/`) and the actual source code simultaneously to prevent them from going out-of-sync.
